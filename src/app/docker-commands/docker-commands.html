Docker Commands

A categorized academic overview of essential Docker CLI commands.

Section 1 — Working with Images

Docker Image Commands

View existing images:

docker images

Download image from Docker Hub:

docker pull <image_name>

Delete image:

docker rmi <image_id>

Build an image from a Dockerfile:

docker build -t myapp .

Section 2 — Handling Containers

Docker Container Commands

Run a container (detached mode):

docker run -d <image_name>

Set an external port:

docker run -d -p 8080:80 <image_name>

Add container name (label):

docker run -d --name myweb <image_name>

Display running containers:

docker ps

Show all containers (including stopped ones):

docker ps -a

Stop container:

docker stop <container_id>

Restart a stopped container:

docker start <container_id>

Delete container:

docker rm <container_id>

Run command inside a container:

docker exec -it <container_id> <command>

Section 3 — Registry Operations

Docker Registry Commands

Login to Docker Hub:

docker login

Search for an image:

docker search <image_name>

Pull an image:

docker pull <image_name>

Push an image:

docker push <image_name>

Note

Detached Mode (-d)

The -d flag runs the container in the background. Use docker ps to see running containers.

Explanation

Port Mapping (-p)

Maps a port inside the container to a host port. Example: -p 8080:80.

Tip

Naming Containers (--name)

Assigns a name to the container for easier reference. Example: docker run -d --name myweb nginx.

Note

Execute Commands Inside Containers

Use docker exec -it <container_id> <command> to run commands inside a running container.

Reminder

Pruning Unused Resources

docker system prune -a removes unused containers, images, and networks. ⚠️ Be careful — this deletes stopped containers and dangling images.

Best Practice

Lightweight Images

Use minimal base images like alpine to reduce size and improve build speed.

Security

Avoid sudo Repetition

Add your user to the docker group: sudo usermod -aG docker $USER. Then log out and back in.

Concept

Image Layers
  • Docker images are built in layers.
  • Layers are cached, speeding up rebuilds.
