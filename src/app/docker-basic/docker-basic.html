Step 1 — Create or Download Image

Creating / Pulling Docker Images

Create a Docker image for your app (via Dockerfile):

docker build -t myapp .

Download (pull) an image from Docker Hub:

docker pull <image-name>

Step 2 — Create Container

Creating Containers

Create a container with specific name, port, and hostname:

docker create --name lab1 -p 82:80 --hostname lab1 <image-name>

This defines the container's name, mapped ports, and hostname before starting it.

Step 3 — Run Container

Running Containers

Run container in detached mode (background):

docker run -d --name lab1 -p 82:80 --hostname lab1 <image-name>

Run container with restart policy:

docker run -d --name lab1 -p 82:80 --hostname lab1 --restart always <image-name>

Run with limited resources:

docker run -d --name lab1 -p 82:80 --hostname lab1 --restart always -m <memory> --cpus <cpus> <image-name>

Flag

-d (Detached Mode)

Runs the container in the background (not directly in the terminal).

Flag

--restart always

Automatically restarts the container whenever it stops or after reboot.

Flag

--name

Assigns a readable name to the container for easier reference.

Flag

-p (Port Mapping)

Maps a container port to a host port, e.g. -p 8080:80.

Flag

--hostname

Defines the hostname of the container.

Flag

Resource Limits (-m, --cpus)

Restricts memory and CPU usage for the container, improving control and efficiency.
