Docker Volumes — Data Persistence

Learn how Docker Volumes help persist and share data between containers safely.

Section 1 — Core Concept

What is a Docker Volume?

A Docker Volume is a special storage mechanism that persists data outside of a container’s writable layer. Volumes are independent of container lifecycle, meaning your data stays safe even if the container is removed.

Section 2 — Visual Overview

Docker Volumes Diagram

Volumes allow containers to read/write persistent data independently of their lifecycle.

Section 3 — Key Docker Volume Commands

Common Commands

docker volume create my_volume

Create a new volume named my_volume.

docker run -d -v my_volume:/app/data my_image

Run a container with my_volume mounted at /app/data.

docker volume ls

List all Docker volumes.

docker volume rm my_volume

Remove a Docker volume (must not be in use).

Section 4 — Practical Example

Shared Volume for Microsoft Windows Server Core

We want to persist shared data in a volume and share it between two containers.

Example Overview

Both win1 and win2 containers mount the same shared_data volume, allowing them to read and write files in the shared directory. This ensures data persistence and easy file sharing across containers. 

# Create a persistent volume for Share Data
docker volume create shared_data

# Run first windows container (win1) using the volume
docker run -d --name win1 -v shared_data:C:\app\data mcr.microsoft.com/windows/servercore:ltsc2019 powershell -Command "Start-Sleep -Seconds 3600"

# Run second windows container (win2) using the same volume
docker run -d --name win2 -v shared_data:C:\app\data mcr.microsoft.com/windows/servercore:ltsc2019 powershell -Command "Start-Sleep -Seconds 3600"

Command

docker volume create

Creates a persistent storage volume independent of containers.

Command

docker run -v

Mounts a volume inside a container for persistent storage.

Command

docker volume ls

Lists all existing volumes in Docker.

Command

docker volume rm

Removes a volume. Must not be in use by any container.

Section 5 — Summary

Key Takeaways

  • Docker Volumes persist data independently of container lifecycle.
  • Volumes can be shared between multiple containers.
  • Use volumes for databases, logs, or any data that must survive container removal.
  • Commands: create, mount (-v), list, and remove volumes.
