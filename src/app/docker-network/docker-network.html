Docker Networking — CNI Overview

Understanding how Docker networks connect containers using CNI.

Section 1 — Core Idea

Main Concept

Docker networking connects containers together and with the outside world using CNI (Container Network Interface).

Section 2 — Visual Overview

Docker Network Structure

Illustration of how Docker connects containers and networks.

Section 3 — CNI Drivers

CNI Drivers

Type Description
Bridge Creates a virtual private network between containers (default).
Host Containers share the host network directly.
Null No network assigned — completely isolated.

Section 4 — Key Commands

Common Network Commands

Create a custom network:

docker network create mynet --subnet 10.10.10.0/24 --gateway 10.10.10.1 --driver bridge

Run container within a network:

docker run -d --name lab1 -p 82:80 --hostname lab1 --restart always --network mynet nginx

Section 5 — Summary

Quick Recap

  • Docker uses CNI to manage container networking.
  • Default mode is Bridge.
  • Custom networks can be created with docker network create.
  • Containers can join networks using --network flag.
