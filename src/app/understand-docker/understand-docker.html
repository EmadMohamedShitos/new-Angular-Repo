Understanding Docker Containers

An academic and practical explanation of Docker and its role in modern computing.

Section 1 — Introduction

What are Containers?

Imagine you have an application (for example, a website or a Python app). This application usually requires:

  • A specific version of Python.
  • Specific libraries.
  • Special system settings.

Setting up this environment on each device is usually time-consuming and error-prone. Containers package the app + dependencies so it runs the same everywhere.

Section 2 — Core Concept

Docker Containers Explained

This is where Docker Containers come in!

A container is an isolated, lightweight runtime environment that contains:

  • Your code.
  • Required libraries.
  • Settings and dependencies.

All within a single, small “box” that can be run anywhere without problems. Docker helps build, ship and run these containers consistently across environments.

Section 3 — Comparison

Container vs Virtual Machine

Section 4 — Mechanism

How Containers Work

  1. Image: Docker builds an image — a template containing everything your app needs.
  2. Container: When executed, the image becomes a running container.
  3. Multiple instances: You can run many containers from the same image; each runs independently.

Section 5 — Practical Example

Practical Example

Run an NGINX web server inside a Docker container:

docker run -d -p 8080:80 nginx

This command downloads the NGINX image, runs it inside a container and maps port 8080 on your host to port 80 in the container. Then open http://localhost:8080.

Note

Quick tips
  • Images are layered — layers are cached.
  • Use small base images to reduce size.

Reminder

Practical step

Use docker ps to list running containers and docker stop <id> to stop them.
