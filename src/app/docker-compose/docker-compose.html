Docker Compose

An academic and practical guide for multi-container Docker applications.

Section 1 — Introduction

What is Docker Compose?

Docker Compose is a tool to define and manage multi-container Docker applications using a single docker-compose.yml file.

Section 2 — Benefits

Why Use Docker Compose?

  • Manage multiple containers at once (web, database, etc.).
  • Define networks and volumes easily.
  • Consistency across different environments.
  • Simple commands to start, stop, and build containers.

Section 3 — Example Overview

Overview of the Example

In this example, we have a simple multi-container application consisting of:

  • Web service: An Nginx container serving static HTML files.
  • Database service: A MariaDB container storing application data.
  • Network: Both containers communicate through app-network.
  • Volumes: Persistent database storage via db_data.

Section 4 — Example docker-compose.yml

Example File

version: "3.9"

services:
  web:
    image: nginx:latest
    ports:
      - "8080:80"
    volumes:
      - ./html:/usr/share/nginx/html
    networks:
      - app-network

  db:
    image: mariadb:latest
    environment:
      MYSQL_ROOT_PASSWORD: root123
      MYSQL_DATABASE: mydb
    volumes:
      - db_data:/var/lib/mysql
    networks:
      - app-network

volumes:
  db_data:

networks:
  app-network:

Section 5 — YAML File Structure

Understanding docker-compose.yml

  • version: Defines the Compose file format version. Here, "3.9".
  • services: Definitions of all containers (e.g., web and db).
    • image: Docker image used for the container.
    • ports: Maps container ports to the host machine.
    • volumes: Mounts directories or named volumes for data persistence.
    • environment: Environment variables for the container.
    • networks: Container networking.
  • volumes: Named volumes for persistent data.
  • networks: Custom networks for container communication.

Section 6 — Commands

Docker Compose Commands

docker-compose up        # Start all containers
docker-compose up -d     # Start containers in background
docker-compose down      # Stop and remove containers, networks, volumes
docker-compose ps        # List running containers
docker-compose logs      # Show logs of all containers
docker-compose build     # Build images defined in the Dockerfile

Section 7 — YAML File Composition

How docker-compose.yml is Structured

The docker-compose.yml file follows a hierarchical structure made up of Sections, Items, and key:value pairs.

  • Section: Top-level category (services, volumes, networks).
  • Item: Name of a specific container or element.
  • key : value pairs: Config details for each item.

Common keys inside each service:

  • image: Docker image used for the container.
  • container_name: Optional custom container name.
  • ports: Maps container ports (e.g., "8080:80").
  • volumes: Attach directories or named volumes.
  • networks: Connect container to networks.

This hierarchical format ensures Docker Compose reads and deploys multi-container setups in a structured and maintainable way.
